# ACSD-68892：可快取頁面的[!DNL Fastly]快取行為不一致

ACSD-68892修補程式修正了[!DNL Fastly]快取中未正確儲存或提供可快取頁面的問題，導致不一致的快取行為並降低效能。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77時，即可使用此修補程式。 修補程式ID為ACSD-68892。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p2

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

未正確儲存可快取頁面，或從[!DNL Fastly]快取提供服務，導致不一致的快取行為並降低效能。

<u>要再現的步驟</u>：

1. 啟用[!DNL Fastly]快取。
1. 建立新的網站/商店/商店檢視。
1. 將次要網站程式碼新增至`index.php`：

```
$_SERVER['MAGE_RUN_TYPE'] = 'website';
$_SERVER['MAGE_RUN_CODE'] = 'web2';
```

1. 使用curl檢查快取控制標頭：

```
curl -Is https://localhost/m247/pub/ | grep cache-control
```

<u>預期結果</u>：

在[!DNL Fastly]中快取可快取頁面。

<u>實際結果</u>：

[!DNL Fastly]中未快取可快取頁面，標頭中傳回`max-age=0`。

```
curl -Is https://localhost/m24develop/pub/ |grep cache-control
cache-control: max-age=0, must-revalidate, no-cache, no-store
```
