# ACSD-69308：在網站層級設定`special_price`時，目錄價格規則未套用

ACSD-69308修補程式修正了僅於網站層級設定`special_price`時（而非在「所有商店檢視」），目錄價格規則不適用的問題。 修正後，會先檢查網站的預設商店，以正確套用目錄價格規則。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76時，即可使用此修補程式。 修補程式ID為ACSD-69308。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

當`special_price`只設定在網站層級時，目錄價格規則不適用。 系統無法正確辨識指派給特定商店檢視之產品的價格，導致價格不一致。

<u>要再現的步驟</u>：

1. 登入「管理」面板。
1. 建立自訂網站、商店和商店檢視(web2)。
1. 將&#x200B;**[!UICONTROL Catalog Price Scope]**&#x200B;設定為&#x200B;*網站*。
1. 將&#x200B;**[!UICONTROL Use for Promo Rule Conditions]**&#x200B;屬性的&#x200B;*設為*&#x200B;是`special_price`。
1. 建立下列型錄價格規則。

```
Website: web2
Conditions:
Special Price  greater than  15
Special Price  less than  50
Actions:
Apply: Apply as percentage of original
Discount Amount: 100
```

1. 建立僅指派給&#x200B;**web2**&#x200B;商店檢視的兩個產品。

產品01：

```
Set Advanced Pricing:
All Store View Scope: 30.00
web2 scope: 40.00
```

產品02：

```
Set Advanced Pricing:
All Store View Scope: keep empty
web2 scope: 40.00
```

1. 重新索引並載入&#x200B;**web2**&#x200B;商店檢視下的店面上的每個產品。

<u>預期結果</u>：

目錄價格規則應套用至Web2商店檢視。 產品02應顯示&#x200B;*0.00*&#x200B;的價格，因為其特殊價格符合目錄規則條件。

<u>實際結果</u>：

* 產品01：價格顯示為&#x200B;*0.00*，符合目錄規則條件。
* 產品02：價格顯示為&#x200B;*40.00*，預期為&#x200B;**不是**，因為目錄規則應該根據網站層級的特殊價格集套用。
