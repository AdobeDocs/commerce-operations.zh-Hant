# ACP2E-4555: Telephone numbers that contain periods or forward slashes don&#39;t validate properly

The ACP2E-4555 patch fixes the issue where modern telephone numbers containing `.` or `/` don&#39;t validate. 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78時，即可使用此修補程式。 The patch ID is ACP2E-4555. 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p3

**與Adobe Commerce版本相容：**

* Adobe Commerce (all deployment methods) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

Modern telephone numbers containing `.` or `/` don&#39;t validate.

<u>要再現的步驟</u>：

1. Log in as a customer on the storefront.
1. Add a new address to the address book with a telephone number, for example: `06.76.40.32.22` or `+43680/2149568`.
1. Save the address details.

<u>預期結果</u>：

您可以搭配句點和正斜線使用現代電話格式。

<u>實際結果</u>：

由於`.`或`/`個字元，電話欄位的驗證失敗。
