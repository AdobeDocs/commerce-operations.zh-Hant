# ACP2E-4565：公司GraphQL查詢傳回&#x200B;*使用X-Adobe-Company標頭時，目前客戶未獲授權*

ACP2E-4565修補程式修正了GraphQL公司查詢傳回錯誤&#x200B;*請求中同時包含X-Adobe-Company和Store標頭時，目前客戶未獲授權*&#x200B;的問題，即使客戶已正確指派給指定的公司和商店。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78時，即可使用此修補程式。 修補程式ID為ACP2E-4565。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p3

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.7-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

公司GraphQL查詢傳回&#x200B;*當請求中同時包含X-Adobe-Company和Store標頭時，目前客戶未獲授權*，即使客戶已正確指派給指定的公司和商店。

<u>要再現的步驟</u>：

1. 啟用B2B公司。
1. 建立其他網站、商店和商店檢視。
1. 建立兩個公司（第一公司和第二公司）。
1. 建立指派給第二個網站的使用者。
1. 將兩個公司指派給使用者。
1. 使用這些標題執行以下GraphQL查詢：
   * `X-Adobe-Company: MQ==`
   * `Store: storeview2`

   ```graphql
   query Company {
     company {
       id
       legal_name
       name
     }
   }
   ```

<u>預期結果</u>：

公司資料成功傳回。

<u>實際結果</u>：

GraphQL回應傳回授權錯誤：

```
{
    "errors": [
        {
            "message": "The current customer isn't authorized.",
            "extensions": {
                "category": "graphql-authorization"
            }
        }
    ]
}
```
