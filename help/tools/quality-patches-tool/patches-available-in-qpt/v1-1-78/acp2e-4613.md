# ACP2E-4613：大型媒體目錄造成媒體集目錄樹狀結構的載入延遲

ACP2E-4613修補程式修正大型媒體目錄結構導致`gettree`回應速度緩慢，導致&#x200B;**[!UICONTROL Media Gallery]**&#x200B;目錄樹狀結構載入時間延長的問題。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78時，即可使用此修補程式。 修補程式ID為ACP2E-4613。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p3

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

大型媒體目錄結構導致`gettree`回應緩慢，導致[!UICONTROL Media Gallery]目錄樹狀結構的載入時間延長。

<u>必要條件</u>：

1. 有許多產品具有多個影像。

<u>要再現的步驟</u>：

1. 產生影像縮圖以增加媒體空間：

   ```
   bin/magento catalog:images:resize
   ```

1. 在`pub/media/catalog/category`下建立多個子目錄（例如：test、test2、test3、test4以及視需要的其他巢狀資料夾）。
1. 將大量目錄媒體檔案複製到新建立的目錄中，以大幅增加媒體目錄樹狀結構的大小，例如：

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. 登入管理面板並瀏覽至&#x200B;**[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**。
1. 請注意，`admin/media_gallery/directories/gettree/?isAjax=true`要求需要很長時間才能完成。

<u>預期結果</u>：

目錄樹狀結構會在幾秒內載入。

<u>實際結果</u>：

視在`pub/media`下的子目錄數目而定，目錄樹狀結構需要幾分鐘的時間載入。
