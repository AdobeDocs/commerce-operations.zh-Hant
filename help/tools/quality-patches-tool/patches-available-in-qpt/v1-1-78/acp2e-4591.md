# ACP2E-4591：根據訂單計數的客戶區段（例如「首次購買者」）不會更新透過REST API所下的訂單

ACP2E-4591修補程式修正了根據訂單計數的客戶區段（例如「首次購買者」）因缺少客戶工作階段而未透過REST API下訂單時進行更新的問題。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78時，即可使用此修補程式。 修補程式ID為ACP2E-4591。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.7-p4

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.4 - 2.4.7-p9

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

透過REST API下訂單時，根據訂單數的客戶區段（例如「首次購買者」）不會更新。

<u>要再現的步驟</u>：

1. 建立名為&#x200B;*第一個使用者*&#x200B;的客戶區段，其條件如下：

   ```
   If ALL of these conditions are TRUE:
   Total Number of Orders less than 1 while ALL of these Conditions match:
   Order Status is Any
   ```

1. 建立客戶。
1. 執行`matchCustomerSegmentProcessor`消費者並確認新客戶出現在&#x200B;**[!UICONTROL Matched Registered Customers]**&#x200B;格線中。
1. 以客戶身分登入，並從店面將產品新增至購物車。
1. 使用REST API端點下訂單。
1. 再次執行相同的消費者。

<u>預期結果</u>：

客戶在下單後會立即從&#x200B;*第一個使用者*&#x200B;區段中移除，而不依賴消費者。

<u>實際結果</u>：

即使下訂單並執行消費者後，客戶仍會留在&#x200B;*第一個使用者*&#x200B;區段中。
