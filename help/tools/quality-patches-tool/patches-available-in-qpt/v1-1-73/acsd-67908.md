# ACSD-67908：在多存放區設定中，JavaScript檔案無法正確合併

ACSD-67908修補程式修正JavaScript檔案無法在多存放區設定中正確合併的問題。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73時，即可使用此修補程式。 修補程式ID為ACSD-67908。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

>[!NOTE]
>
>Adobe不建議使用&#x200B;**[!UICONTROL Merge JavaScript Files]**。 在套用此修補程式之前，請嘗試停用JavaScript合併並重新部署靜態內容以確認問題是否已解決。 如需詳細資訊，請參閱[最佳化資源檔](/help/implementation-playbook/best-practices/development/optimize-css-js-files.md#merge-js-files)。


## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.7-p5

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

合併的JavaScript檔案無法在多存放區設定中正確產生，導致檔案損毀和存放區錯誤。

<u>要再現的步驟</u>：

1. 設定多個網站、商店和商店檢視。
1. 設定生產模式並略過編譯：

   ```shell
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. 部署產生的程式碼：

   ```shell
   bin/magento setup:di:compile
   ```

1. 使用精簡策略部署靜態內容：

   ```shell
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. 刪除合併快取資料夾的內容：

   ```text
   rm -rf pub/static/_cache/merged/*
   ```

1. 清除快取：

   ```shell
   bin/magento cache:clean
   ```

1. 在個別的瀏覽器分頁或視窗中同時開啟所有商店檢視。


<u>預期結果</u>：

所有JavaScript檔案已成功合併，店面功能正常，沒有錯誤。

<u>實際結果</u>：

部分合併的JavaScript檔案已損毀，且開頭不是正確的[!DNL RequireJS]模組宣告。
