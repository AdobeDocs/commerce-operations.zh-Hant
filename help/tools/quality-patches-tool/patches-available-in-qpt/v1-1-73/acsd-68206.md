# ACSD-68206： [!DNL GraphQL]伺服器因速率限制和PHP Redis延伸而失敗

ACSD-68206修補程式修正了使用「速率限制」功能和PHP Redis擴充功能時，GraphQL應用程式伺服器擲回錯誤，進而中斷速率限制功能的問題。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73時，即可使用此修補程式。 修補程式ID為ACSD-68206。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p2

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

當GraphQL應用程式伺服器使用速率限制功能並安裝PHP Redis擴充功能時，就會發生此問題，導致速率限制功能失效。

<u>要再現的步驟</u>：

1. 安裝PHP Redis擴充功能。
1. 設定應用程式的Redis。
1. 設定費率限制器，允許每分鐘為每位客戶訂購一單。
1. 啟動[!DNL GraphQL]應用程式伺服器。
1. 作為客戶，請使用`PlaceOrder` [!DNL GraphQL]突變提交訂單。
1. 在一分鐘內，使用`PlaceOrder` [!DNL GraphQL]突變為相同客戶提交第二份訂單。

<u>預期結果</u>：

`PlaceOrder` [!DNL GraphQL]要求傳回訊息限制訂單嘗試。

<u>實際結果</u>：

伺服器針對第二個PlaceOrder GraphQL請求傳回500錯誤。 例外記錄記錄：

```
report.CRITICAL: Error: Call to a member function exec() on null in vendor/colinmollenhour/credis/Client.php:1336
```
