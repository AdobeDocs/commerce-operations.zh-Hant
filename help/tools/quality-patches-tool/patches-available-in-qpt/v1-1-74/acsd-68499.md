# ACSD-68499：GraphQL updateCartItems突變傳回&#x200B;*success*，數量超過可用存量

ACSD-68499修補程式修正了在更新數量超過可用庫存量時，GraphQL `updateCartItems`突變傳回不正確&#x200B;*success*&#x200B;回應的問題，進而導致數量與總數膨脹。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74時，即可使用此修補程式。 修補程式ID為ACSD-68499。 請注意，此問題已排程在Adobe Commerce 2.4.9中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） 2.4.8-p2

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

當更新數量超過可用庫存時，Adobe Commerce中的GraphQL `updateCartItems`突變傳回&#x200B;*success*&#x200B;回應不正確，導致購物車總計和數量膨脹。

<u>要再現的步驟</u>：

1. 建立空的購物車。
1. 將庫存有限的產品（例如10件可用）新增到購物車。
1. 查詢購物車並記下`cart_item_id`。
1. 使用以下變異，將購物車料號數量更新為100 （超過10的可用存量）：

   ```
   mutation ($cart: String!, $cartItemId: Int!, $quantity: Float!) {
     updateCartItems(
       input: {
         cart_id: $cart
         cart_items: [{ cart_item_id: $cartItemId, quantity: $quantity }]
       }
     ) {
       cart {
         items {
           product {
             name
           }
           quantity
         }
         prices {
           grand_total {
             value
             currency
           }
         }
       }
     }
   }
   ```

1. 再次查詢購物車以驗證實際數量。

<u>預期結果</u>：

庫存驗證可防止無效的數量更新。

<u>實際結果</u>：

`updateCartItems`突變傳回數量為100的&#x200B;*success*&#x200B;回應。
