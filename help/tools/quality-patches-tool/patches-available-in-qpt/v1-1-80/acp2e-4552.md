# ACP2E-4552： GraphQL回應未傳回公司狀態

ACP2E-4552修補程式修正GraphQL回應中未傳回公司狀態的問題。 安裝[[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80時，即可使用此修補程式。 修補程式ID為ACP2E-4552。 請注意，此問題已排程在Adobe Commerce B2B 1.5.2-p3中修正。

## 受影響的產品和版本

**已為Adobe Commerce版本建立修補程式：**

* Adobe Commerce （所有部署方法） B2B 1.5.2-p3

**與Adobe Commerce版本相容：**

* Adobe Commerce （所有部署方法） B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>此修補程式可能適用於發行版本為[!DNL Quality Patches Tool]的其他版本。 若要檢查修補程式是否與您的Adobe Commerce版本相容，請將`magento/quality-patches`套件更新至最新版本，並在[[!DNL Quality Patches Tool]上檢查相容性：搜尋修補程式頁面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=zh-Hant)。 使用修補程式ID作為搜尋關鍵字，以尋找修補程式。

## 問題

GraphQL回應不會傳回公司狀態。

<u>必要條件</u>：

[!DNL B2B]已安裝。

<u>要再現的步驟</u>：

1. 在Admin中，移至&#x200B;**[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]**，並將&#x200B;**[!UICONTROL Enable Company]**&#x200B;設為`Yes`。
1. 以管理員身分登入Commerce管理員。
1. 建立具有以下狀態的三個公司：有效、已拒絕及未決核准。
1. 在Admin中，移至&#x200B;**[!UICONTROL Customers]** > **[!UICONTROL All Customers]**。 開啟客戶帳戶，並從&#x200B;**[!UICONTROL Associate to Company]**&#x200B;清單中，將客戶指派給每個已建立的公司。
1. 傳送下列GraphQL請求：

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>預期結果</u>：

GraphQL回應會傳回與客戶相關聯之所有公司的正確狀態值（例如「作用中」、「未決核准」、「已拒絕」及「已封鎖」）。 您也可以依公司狀態篩選或排序回應。

<u>實際結果</u>：

GraphQL回應不包含與客戶相關聯之公司的公司狀態值。
