# Adobe Commerce字彙表

## A

### 在折頁上方

_形容詞_

在瀏覽器視窗中，載入網頁後和使用者向下捲動頁面之前立即顯示的內容。
在設計版面時，請使用彈性的格式，以最佳方式顯示此區域最優先的產品、功能、銷售、通知、選項等。

透過行動裝置和平板電腦，折頁上方的區域差異很大，尤其是在檢視時螢幕的大小和尺寸以及方向（縱向與橫向）上。
使用回應式主題和測試有助於尋找內容和版面的適當組合。

_詞語屬性：_

* _欄位：設計_

### 活動分支

_名詞_

活動分支或環境是連接到已部署或正在運行且具有服務訪問權限的實例的分支或環境。
停用時，會移除與服務和執行中執行個體的連線，但會保留程式碼。
它變成了普通的git分支。

_詞語屬性：_

* _欄位：雲端_

### 適配器

_名詞_

一種類，它使兩個其他不相容的系統能夠協同工作，而不修改系統的原始碼。
示例包括資料庫適配器、快取適配器、檔案系統適配器、後處理器庫適配器和其他類型的計算適配器。

* _欄位：寫程式_

### 區域

_名詞_

區域是Magento應用範圍的抽象術語。
區域是邏輯元件，可組織程式碼以進行最佳化的請求處理。
區域減少了從店面訪問的配置對象的記憶體需求，並通過僅載入所需的相關代碼來簡化Web服務調用。
每個區域都可包含完全不同的程式碼，以處理URL和請求。

Adobe Commerce區域包括：

* 管理員(adminhtml)
* 店面
* Web API REST(webapi_rest)

_詞語屬性：_

* _欄位：商務軟體_
* _相關術語：商務元件，商店_

### 屬性

_名詞_

描述產品某些方面的產品的特性或屬性。
Adobe Commerce或Magento Open Source使用者可以建立自訂屬性，以新增至預設屬性集或自訂屬性集。
透過管理員或以程式設計方式建立這些屬性。
範例：顏色、大小、重量、價格、年齡、性別等。

自訂屬性是實體屬性值(EAV)屬性的類型。

針對Google購物廣告管道和AmazonSales Channel等整合，您可將商務屬性對應至協力廠商中的屬性，以正確顯示和銷售產品、顯示廣告。

更多詳情： [EAV和擴充功能extension_attributes](https://developer.adobe.com/commerce/php/development/components/attributes/)

* _欄位：商務軟體，產品_
* _同義詞：產品屬性，自訂屬性_
* _相關術語：延伸功能屬性_

### 目錄

_名詞_

對於商戶，目錄代表其產品詳細目錄。
在Adobe Commerce架構中，目錄包含類別、產品和產品屬性。

每個商務商店只有一個產品目錄，由所有商店檢視共用。
在多商店安裝中，每個商店可以有單獨的目錄，或者共用目錄。
目前的商店目錄由預設的根類別決定，該類別在商店的頂端導覽（主功能表）中對使用者可見。
（「根類別」一詞可能令人困惑，因為「根類別」實際上並不是類別，而是功能表的容器，由類別和子類別組成。）

您可以建立任意數量的根類別，但一次只能使用一個（預設值）。

### 可配置產品

_名詞_

可設定的產品看起來就像單一產品，包含每個變異的選項下拉式清單。
每個選項實際上都是個別的簡單產品，具有獨特的SKU，因此可追蹤每個產品變異的詳細目錄。
為了達到類似的效果，簡單的產品可與自訂選項搭配使用，但無法追蹤每個變異的詳細目錄。
具有多個選項的產品有時稱為複合產品。

雖然可配置產品使用的SKU較多，而且最初設定所需時間可能較長，但最終可以節省您的時間。
如果您打算發展業務，可設定的產品類型對於具有多種選項的產品可能是更好的選擇。

範例：有兩種顏色和三種尺寸的T恤。
必須以個別產品的形式建立6種變體（每種產品都有其自己的SKU）。
然後，所有變體都會新增至可設定的產品，客戶可在此選擇大小和顏色，然後將其新增至購物車。

_詞語屬性：_

* _欄位：商務軟體，產品_
* _相關術語：產品類型_

### 向上

PWA專案可包含UPPRAD定義檔案，以指定其服務相依性。
