### 區域

_名詞_

區域是Magento應用程式範圍的抽象術語。
區域是邏輯元件，可組織程式碼以進行最佳化的請求處理。
區域會減少從店面存取之設定物件的記憶體需求，而且會透過僅載入必要的相依程式碼來簡化Web服務呼叫。
每個區域可以包含完全不同的程式碼，以處理URL和請求。

Adobe Commerce區域包括：

* 管理員(adminhtml)
* 店面
* Web API REST (webapi_rest)

### 屬性

_名詞_

描述產品某些層面的產品特性或屬性。

### 可設定的產品

_名詞_

可設定的產品看起來像是單一產品，每個變數都有選項下拉式清單。
每個選項實際上都是個別的簡單產品，具有唯一的SKU，因此可以追蹤每個產品變體的存貨。
為了達到類似的效果，簡單的產品可以與自訂選項一起使用，但無法追蹤每個變數的詳細目錄。
具有多個選項的產品有時稱為複合產品。

雖然可設定的產品會使用較多SKU，且一開始可能需要較長時間進行設定，但最終可為您節省時間。
如果您計畫拓展業務，對於有多重選項的產品來說，可設定的產品型別可能是更好的選擇。

範例：有兩種顏色和三種尺寸的T恤。
六個變體必須建立為個別產品（每個產品都有自己的SKU）。
然後，所有變體都會新增至可設定的產品，客戶可在其中選擇大小和顏色，然後將其新增至購物車。

_字詞屬性：_

* _欄位：商務軟體、產品_
* _相關詞語：產品型別_

### 共用目錄

_名詞_

一種功能，可讓商家建立可作為完整目錄或目錄子集的目錄，然後指定一或多個產品的自訂價格。
然後，商家可以指派此目錄給一或多個公司。

例如，一個B2B商家有三個客戶議定了商家電子分銷網站的特定費率。
使用共用目錄功能，商傢俱有：

* 主要目錄
* 客戶1目錄（可能只有三個SKU，從主要目錄提供大幅折扣）
* 客戶2目錄（可能是整個目錄，有10%的折扣）
* 客戶3目錄（主要目錄折扣為5%至60%的幾十種產品）。

_字詞屬性：_

* _欄位：商務軟體、產品_
* _相關詞語：目錄、b2b_

### 向上

[PWA Studio](https://github.com/magento/pwa-studio) 使用 [向上](https://developer.adobe.com/commerce/pwa-studio/guides/packages/upward/) 開發中。
UPLOAD是Unified Progressive Web App回應定義的縮寫。
UPLOAD定義檔說明網頁伺服器如何傳遞及支援Progressive Web Application。

UPWARD定義檔案使用平台無關的宣告式語言提供伺服器行為的詳細資訊。
這允許Progressive Web Application在任何技術棧疊上的任何語言中在符合UPPER的伺服器上執行，因為應用程式只關心來自UPPER伺服器的HTTP端點行為。

UPLOAD伺服器使用定義檔案來決定應用程式Shell要求的適當程式或服務。
它說明伺服器應如何處理請求並為其建立回應。

PWA專案可以包含UPLOAD定義檔案來指定其服務相依性。

* _欄位：設計、商務軟體、程式設計_

### 虛擬型別

_名詞_

虛擬型別是一種將不同相依性插入現有PHP類別的方法，而不會影響其他類別，也不必建立類別檔案。
虛擬型別只能由中的引數覆寫參照 `<type>` 元素（在di.xml檔案中），絕對不要在原始程式碼中。
