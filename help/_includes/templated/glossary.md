# Adobe Commerce辭彙表

## A

### 在折疊上方

_形容詞_

在瀏覽器窗口中，在載入網頁後和用戶向下滾動頁面之前立即可見的內容。
在設計佈局時，使用靈活的格式來最好地顯示此區域中優先順序最高的產品、功能、銷售、通知、選項等。

借助移動和平板電腦，折疊上方的面積差別很大，特別是在螢幕尺寸和尺寸以及觀看時的方向（縱向和橫向）上。
使用響應性主題和測試可幫助找到內容和佈局的適當組合。

_術語屬性：_

* _欄位：設計_

### 活動分支

_名詞_

活動分支或環境是連接到具有服務訪問權限的已部署或正在運行的實例的分支或環境。
停用時，將刪除與服務和運行實例的連接，但將保留代碼。
它變成了一個普通的Git分支

_術語屬性：_

* _欄位：雲_

### 適配器

_名詞_

一個類，它使兩個其他不相容的系統能夠協同工作，而不修改系統的原始碼。
示例包括資料庫適配器、快取適配器、檔案系統適配器、後處理器庫適配器和其他類型的計算適配器。

_術語屬性：_

* _欄位：寫程式_

### 管理員

_名詞_

在軟體中，具有完全管理員權限的用戶角色可管理所有功能。
在Adobe Commerce，管理員用戶擁有對管理員中所有功能、選項和權能的完全權限和訪問權限。
它們還可以建立用戶和角色。

瞭解詳情： [添加用戶](https://experienceleague.adobe.com/docs/commerce-admin/systems/user-accounts/permissions-users-all.html)

_術語屬性：_

* _欄位：商務軟體_
* _同義詞：管理員，超級用戶_
* _相關條款：商務管理員_

### 管理區

_名詞_

您商店的受密碼保護的後台辦公室，在該辦公室中管理訂單、目錄、內容和配置。
用戶訪問管理區以管理商店，包括產品、訂單、發運、CMS內容、店面設計、客戶資訊等。
管理員用戶具有控制對功能、選項和權能的訪問權限的關聯角色。

瞭解詳情： [Adobe Commerce使用手冊](https://experienceleague.adobe.com/docs/commerce-admin/user-guides/home.html)

_術語屬性：_

* _欄位：商務軟體_
* _同義詞：管理員、管理面板、後端、管理介面、管理員UI_
* _相關條款：管理員_

### 面積

_名詞_

區域是Magento適用範圍的抽象術語。
區域是組織代碼以進行優化請求處理的邏輯元件。
區域減少了從儲存前端訪問的配置對象的記憶體需求，並通過僅載入所需的相關代碼來簡化Web服務調用。
每個區域都可以包含處理URL和請求的完全不同的代碼。

Adobe Commerce地區包括：

* 管理員(adminhtml)
* 店面
* Web API REST(webapi_rest)

_術語屬性：_

* _欄位：商務軟體_
* _相關條款：商務元件，店面_

### 屬性

_名詞_

描述產品某些方面的產品的特徵或屬性。
Adobe Commerce或Magento Open Source用戶可以建立自定義屬性以添加到預設屬性集或自定義屬性集。
通過管理或寫程式方式建立這些屬性。
示例：顏色、大小、重量、價格、年齡、性別等。

自定義屬性是「實體 — 屬性 — 值」(EAV)屬性的類型。

對於Google購物廣告渠道和AmazonSales Channel等整合，您可以將Commerce屬性映射到第三方的屬性，以正確顯示和銷售產品、顯示廣告。

瞭解詳情： [EAV和擴展擴展屬性](https://developer.adobe.com/commerce/php/development/components/attributes/)

* _欄位：商業軟體，產品_
* _同義詞：產品屬性，自定義屬性_
* _相關條款：擴展屬性_

### 目錄

_名詞_

對於商家，目錄表示其產品庫存。
在Adobe Commerce體系結構中，目錄由類別、產品和產品屬性組成。

每個Commerce商店只有一個產品目錄，所有商店視圖都共用該目錄。
在多儲存區安裝中，每個儲存區都可以有單獨的目錄，或共用目錄。

（術語「根類別」可能令人困惑，因為「根類別」根本不是真正的類別，而是包含類別和子類別的菜單的容器。）

### 可配置產品

_名詞_

可配置產品看起來像是單個產品，每個變體的選項下拉清單。
每個選項實際上都是一個單獨的簡單產品，具有唯一的SKU，因此可以跟蹤每種產品的庫存變化。
為獲得類似效果，可將簡單產品與定製選項一起使用，但無法跟蹤每種變化的庫存。
具有多個選項的產品有時稱為複合產品。

儘管可配置產品使用的SKU更多，而且初始設定可能需要稍長的時間，但它最終可以節省您的時間。
如果您計畫發展業務，可配置的產品類型可能是具有多個選項的產品的更好選擇。

示例：有兩種顏色和三種尺寸的T恤。
必須將六種變型建立為單個產品（每種產品都有自己的SKU）。
然後，所有變型都會添加到可配置產品中，客戶可以在其中選擇大小和顏色，然後將其添加到購物車中。

_術語屬性：_

* _欄位：商業軟體，產品_
* _相關條款：產品類型_

### 共用目錄

_名詞_

一種功能，允許商家建立一個目錄，該目錄可以作為其整個目錄或其子集，然後為一個或多個產品指定自定義價格。
然後，商家可以將此目錄分配給一個或多個公司。

例如，B2B商戶有三個客戶，他們已經為該商戶的電子分銷站點協商了特定費率。
使用共用目錄功能，商戶具有：

* 主目錄
* 客戶1目錄（可能只是3個SKU，在主目錄中有較高折扣）
* 客戶2目錄（可能是整個目錄，10%的折扣）
* 客戶3目錄（幾十種產品，主目錄的折扣範圍在5%到60%之間）。

_術語屬性：_

* _欄位：商業軟體，產品_
* _相關條款：目錄，b2b_

### 向上

_名詞_

[PWA Studio](https://github.com/magento/pwa-studio) 使用 [向上](https://developer.adobe.com/commerce/pwa-studio/guides/packages/upward/) 開發。
OWPRAMD是Unified Progressive Web App響應定義的縮寫。
OWPRAMD定義檔案描述Web伺服器如何傳遞和支援Progressive Web Application。

OWDRAM定義檔案使用獨立於平台的聲明性語言提供有關伺服器行為的詳細資訊。
這允許Progressive Web Application在任何技術堆棧上以任何語言在符合OWPRAMD的伺服器上運行，因為應用程式只關心來自OWPRAMD伺服器的HTTP端點行為。

OWPRAMD伺服器使用定義檔案來為來自應用程式外殼的請求確定適當的進程或服務。
它描述了伺服器應如何處理請求並為其生成響應。

PWA項目可以包括OWPRAMD定義檔案以指定其服務相關性。

_術語屬性：_

* _欄位：設計，商業軟體，寫程式_
* _同義詞：PWA Studio，統一累進Web App響應定義_
* _相關條款：普瓦_
